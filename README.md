# 🛡️ Система предотвращения мошенничества с банковскими картами в реальном времени

**Дипломный проект** — многокомпонентная система обнаружения финансового мошенничества на основе ансамблевого машинного обучения.

Система развёрнута на 3 ноутбуках в локальной сети с реальным временем обработки транзакций, генерацией реалистичных данных через Claude API и визуализацией на интерактивном дашборде.

## 🚀 Установка и запуск

### Требования

- **Python 3.12+**
- **SQLite 3** (встроена в Python, никакой установки не нужна! ✅)
- **Node.js 18+** (для frontend)
- **Git**
- **Groq API key** (бесплатный! https://console.groq.com/keys)

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd new_diplom
```

### Шаг 2: Активация виртуального окружения

**Linux/Mac:**

```bash
source venv/bin/activate
```

### Шаг 3: Установка зависимостей Python

```bash
pip install -r requirements.txt
```

### Шаг 4: Настройка переменных окружения

```bash
cp .env.example .env
```

**Отредактируй .env (только одна строка нужна!):**

```bash
# Замени xxx на свой Groq API key (бесплатный!)
GROQ_API_KEY=
```

**Как получить Groq API key (5 секунд):**

1. Открой https://console.groq.com/keys
2. Нажми "Create API Key"
3. Скопируй ключ в .env

**Что уже настроено автоматически:**

- ✅ `DATABASE_URL=sqlite:///./fraud_detection.db` — БД создаётся сама!
- ✅ Fraud score thresholds (0.4/0.7/1.0) — оптимально
- ✅ MAIN_SERVER_HOST — если нужна локальная сеть, отредактируй IP

### Шаг 5: Инициализация БД (автоматическая!)

```bash
# БД создаётся автоматически при первом запуске!
# Никаких миграций не требуется для SQLite ✅
```

### Шаг 6: Запуск backend сервера

```bash
# БД создаётся автоматически в fraud_detection.db
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

Сервер будет доступен: `http://localhost:8000`
API документация: `http://localhost:8000/docs`

### Шаг 7: Запуск frontend (в отдельном терминале)

```bash
cd frontend
npm install
npm start
```

Дашборд будет доступен: `и live`

## 🏛️ Архитектура системы

### Трёхуровневая архитектура на 3 ноутбуках

```
Ноутбук 1 (Главный сервер)
├─ FastAPI сервер (8000)
├─ PostgreSQL БД
├─ ML движок (XGBoost, Random Forest, LSTM, etc.)
├─ WebSocket для real-time
└─ React Dashboard

Ноутбук 2 (Клиент A)
└─ Отправитель транзакций → REST API → Ноутбук 1

Ноутбук 3 (Клиент B)
└─ Получатель транзакций → REST API → Ноутбук 1
```

### ML Ансамбль (5 моделей)

| Уровень | Модель              | Назначение                            |
| ------- | ------------------- | ------------------------------------- |
| **1**   | XGBoost             | Транзакционные паттерны               |
| **1**   | Random Forest       | Поведенческие аномалии                |
| **1**   | LSTM                | Временные последовательности          |
| **2**   | Isolation Forest    | Сетевые/статистические аномалии       |
| **2**   | Rule Engine         | Жесткие правила (ночь + VPN = риск)   |
| **3**   | Logistic Regression | Мета-обучение → финальный FRAUD SCORE |

### Пороги принятия решения

| FRAUD SCORE | Решение          | Статус  |
| ----------- | ---------------- | ------- |
| 0.0 – 0.4   | ✅ ОДОБРЕНО      | Зелёный |
| 0.4 – 0.7   | ⚠️ НА ПРОВЕРКУ   | Жёлтый  |
| 0.7 – 1.0   | ❌ ЗАБЛОКИРОВАНО | Красный |

## 🎯 12 паттернов мошенничества

Система обнаруживает следующие паттерны:

1. ✅ Регулярные поступления от разных людей (как бизнес)
2. ✅ Одинаковые суммы от множества людей
3. ✅ Частые ночные переводы (02:00–05:00)
4. ✅ Резкий рост оборота vs 30-дневная история
5. ✅ Крупные переводы между одними людьми
6. ✅ Дробление суммы (структурирование)
7. ✅ Неофициальный бизнес получает оплаты как магазин
8. ✅ Частые международные переводы
9. ✅ Деньги сразу снимаются наличкой
10. ✅ VPN/TOR + невпопадание геолокации
11. ✅ Невозможное перемещение (KZ→DE за 2 мин)
12. ✅ Velocity-атака (50+ транзакций за 10 мин)

## 🤖 Генератор данных (Claude API)

Система автоматически генерирует реалистичные транзакции каждые **10 секунд**:

- Реалистичные имена, ID, страны, банки
- Случайные суммы, валюты, времена
- Смесь нормальных и мошеннических паттернов
- AI объяснения решений на понятном языке
